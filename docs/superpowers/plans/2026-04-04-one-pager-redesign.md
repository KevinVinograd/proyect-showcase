# One-Pager Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the current generic portfolio one-pager with a narrative-driven site using shadcn/ui, a dual-font system, and tokenized design system.

**Architecture:** Full replacement of App.tsx monolith. Each section becomes its own component in `src/components/sections/`. shadcn/ui provides button, card, badge primitives. All styling via Tailwind + CSS custom properties (design tokens). framer-motion for scroll reveals only.

**Tech Stack:** React 19, Vite 8, TypeScript, Tailwind CSS 4, shadcn/ui, framer-motion, lucide-react

**Spec:** `docs/superpowers/specs/2026-04-04-one-pager-redesign-design.md`

**Files:**

- Modify: `vite.config.ts`

**Context:** shadcn/ui with Tailwind CSS v4 requires specific setup. We need the `clsx` + `tailwind-merge` utility, path aliases, and the shadcn config file. Since Tailwind v4 uses CSS-based config (not tailwind.config.js), shadcn init may need manual adjustments.

- [ ] **Step 1: Install shadcn dependencies**

Run:
```bash
npm install class-variance-authority clsx tailwind-merge
```

Expected: packages added to `dependencies` in package.json

- [ ] **Step 2: Add path alias to tsconfig.app.json**

Add `baseUrl` and `paths` to `compilerOptions`:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

Keep all existing options. Only add these two.

- [ ] **Step 3: Add path alias to vite.config.ts**

```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'
import path from 'path'

export default defineConfig({
  plugins: [react(), tailwindcss()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

- [ ] **Step 4: Create `src/lib/utils.ts`**

```typescript
import { type ClassValue, clsx } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

- [ ] **Step 5: Create `components.json`** (shadcn config)

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "rsc": false,
  "tsx": true,
  "tailwind": {
    "config": "",
    "css": "src/index.css",
    "baseColor": "zinc",
    "cssVariables": true
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib"
  }
}
```

- [ ] **Step 6: Verify build works**

Run: `npx tsc -b && npx vite build`

Expected: Build succeeds with no errors. The old App.tsx still renders — we haven't touched it yet.

- [ ] **Step 7: Commit**


```tsx
import { Team } from "@/components/sections/team"
import { Contact } from "@/components/sections/contact"
import { Footer } from "@/components/sections/footer"

function Reveal({ children }: { children: ReactNode }) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      whileInView={{ opacity: 1, y: 0 }}
      viewport={{ once: true, margin: "-100px" }}
      transition={{ duration: 0.5, ease: "easeOut" }}
    >
      {children}
    </motion.div>
  )
}

export default function App() {
  return (
    <>
      <Hero />
      <div className="max-w-[820px] mx-auto px-[var(--sp-6)]">
        <Reveal><Problem /></Reveal>
        <Reveal><WhatWeBuild /></Reveal>
        <Reveal><Process /></Reveal>
        <Reveal><Portfolio /></Reveal>
        <Reveal><Technologies /></Reveal>
        <Reveal><Team /></Reveal>
        <Reveal><Contact /></Reveal>
        <Footer />
      </div>
    </>
  )
}
```
